## Slice 2 — Interactive self-sustained REPL

### New files
- **`src/config.js`** — global user config (`~/.lazyglm/config.json`): load/save, chmod 600, cached. Shape `{ provider, api_key, model, onboarded }`.
- **`src/onboard.js`** — first-run onboarding: detect missing config → prompt provider (default `zai`) + API key → write `~/.lazyglm/config.json` (chmod 600) via readline.
- **`src/repl.js`** — interactive REPL loop. Reuses `provider.chat` (streaming), `Context`, `TOOL_SPECS`/`TOOL_HANDLERS`, `HookEngine` + `loadPlugins()`, `detectSkillInvocation`/`getSkill`. Does **not** call `runAgent`.
- **`src/sessions.js`** — session persistence: JSONL to `~/.lazyglm/sessions/<id>.jsonl`; list / load / continue.

### Config layering (security-critical)
Provider resolution priority (high → low):
1. CLI flags (`--provider`, `--model`)
2. Env vars (`LAZYGLM_PROVIDER`, `LAZYGLM_API_KEY`, `LAZYGLM_BASE_URL`)
3. `~/.lazyglm/config.json` (`api_key`, `provider`, `model`)
4. catalog default (`zai`)

**The API key never enters `process.env`.** The config-file key is threaded explicitly through `resolveProviderConfig` → `chat()`'s `config.apiKey`. This matters because `run_shell` inherits `process.env` — if the key were in env, the GLM agent could exfiltrate it via `echo $LAZYGLM_API_KEY`. Env vars still work (Daniel's existing export); config-file keys stay out of env.

Existing-file changes: `resolveProviderConfig` + `resolveProvider` + `pickModel` gain an optional `userConfig` param (from `loadUserConfig()`, cached, `{}` when absent). Existing tests that call `pickModel` without it keep working.

### REPL turn loop
```
launch() → loadConfig (or onboard) → ensure project init → open session → REPL
each turn:
  read user input (readline)
  if "/" slash command → dispatch (/help /exit /clear /model /cost /compact /resume /ultrawork /yolo)
  if "$skill" invocation → expand skill body into the user message
  else:
    fire UserPromptSubmit hook → push user msg
    stream chat(model, ctx.messages, TOOL_SPECS) → render reasoning/text/tool deltas
    if tool_calls → execute each (Pre/PostToolUse hooks), push tool results, loop again (agent sub-turn)
    if text-only (no tool) → print, return control to user (NO auto-nudge / no auto-finish)
```
**Key difference from `runAgent`:** the REPL is human-in-the-loop. A text-only response ends the agent turn and returns control to the user — it does **not** nudge for `finish` or auto-stop. The agent only loops on tool calls within a single user turn (so "create hello.js and run it" completes in one user turn via `write_file` → `run_shell`). `finish` in REPL = end turn gracefully (print summary, return to user). Agent sub-turn budget per user turn: ~40.

### Agent persona (REPL system prompt)
The provided LazyGLM persona (lazy pragmatic dev; `patch_file` over whole files; autonomous file lookup; verify with `run_shell`; concise; no "Certainly!" filler) is the REPL system prompt, plus the ENVIRONMENT block (cwd/git/model/date) + hook injects (SessionStart).

### Permissions
- Default: **AUTO** — execute every tool, report after (like Hermes).
- `--yolo` / `/yolo`: explicit "I trust this run" toggle; same as auto this slice (no per-action approval this slice). Reserved for a future per-action mode.

### Sessions
- `~/.lazyglm/sessions/<id>.jsonl` — append every event (user/assistant/tool/usage) + a header record with session meta (id, model, provider, startedAt, firstPrompt).
- `lazyglm --continue` → load most-recent session, replay history into `Context`.
- `/resume` → list past sessions (id, first prompt, mtime); pick one; load.
- Plain `lazyglm` → new session.

### Slash commands
`/help` `/exit` `/clear` `/model <name>` `/cost` `/compact` `/resume` `/ultrawork "<task>"` `/yolo` (toggles). Plus inline `$skill` expansion.

### `/ultrawork` in REPL
Calls `runUltrawork({ task, cwd, model, config, completionPromise, verifyCommand, maxIterations, onEvent: <REPL stream renderer> })`. Streams each iteration. Reports verify PASS/FAIL into the REPL. Runs as a sub-loop (fresh agent context per iteration, like `lazyglm run --ultrawork`), then returns control to the REPL.

### Auto-init
On REPL launch, if cwd lacks `.lazyglm/` or `AGENTS.md`, run `install({ cwd })` silently (creates `.lazyglm/` + `AGENTS.md` + `.gitignore` entry) before first interaction.

### CLI change (`src/cli.js`)
`lazyglm` (no command) OR `lazyglm chat` → launch REPL. Flags: `--continue`, `--yolo`, `--model <name>`, `--provider <name>`. Keep `run`, `doctor`, `models`, `skills`, `install`, `skill`, `hook` as-is.
